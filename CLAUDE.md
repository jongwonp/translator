## 응답 언어

- 사용자에게는 한국어로 설명한다.

## 작업 원칙

- 변경은 항상 최소 범위로 한다.
- 기존 패턴과 기존 유틸을 우선 재사용한다.
- 관련 없는 코드까지 함께 리팩터링하지 않는다.
- 단순한 문제를 해결하기 위해 불필요한 라이브러리를 추가하지 않는다.
- 현재 단계에 꼭 필요하지 않은 고급 구조는 도입하지 않는다.
- 요구사항이 애매할 때는 현재 프로젝트 단계에 맞는 가장 단순하고 실용적인 구현을 우선한다.
- 이미 잘 동작하는 코드를 스타일만을 이유로 다시 쓰지 않는다.
- 커밋은 한 기능 단위로 사용자가 직접 시행한다.
- 사용자가 여러가지 기능에 대해 한 번에 물어보는 상황에서도 커밋을 할 수 있도록 한 기능 단위로 완성하고 끊어준다.
